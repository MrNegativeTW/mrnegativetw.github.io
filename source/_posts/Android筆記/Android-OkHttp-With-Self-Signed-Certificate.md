You got a mission: "add a self-signed certificate to an Android app target below API 24"

You searched. You found the documents from Google. But it only works above API 24.

"How about adding it to the OkHttp client"  - You think

<!--more-->

> If your app's target is above Android 7 (API 24), check the [Android Developers Documentation](https://developer.android.com/training/articles/security-config#manifest) instead.

## Keytool

Print the details of the certificate:

```bash
$ openssl x509 -inform der -in ca.cer -noout -text
```

```
Certificate:
    Data:
        Version: 3 (0x2)
        Serial Number:
            32:a4:df:2e:76...
            ...
            ...
            ...略...
```

The certificate file's extension from the server side may be one of the following: `.crt`, `.pem`, `.cer`, `.key`

Prints the content of the certificate using the following command (In my case I got `.cer`)

```bash
$ keytool -printcert -rfc -file YOUR_CA.cer
```

## Dependencies

Back to Android Studio, and add the dependencies in your `build.gradle(app)`

```gradle
dependencies {
    // define a BOM and its version
    implementation(platform("com.squareup.okhttp3:okhttp-bom:4.10.0"))

    implementation("com.squareup.okhttp3:okhttp")
    implementation("com.squareup.okhttp3:okhttp-tls")
}
```
